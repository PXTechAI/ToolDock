# ToolDock

**日々の開発作業を支える、ローカル動作のデスクトップツールボックス。**

[English](README.md) | [简体中文](README.zh-CN.md) | **日本語**

ToolDock は、画面上の色取得、ポートプロセス管理、スクリーンショット、画面録画、安全な文字列生成を一つにまとめたデスクトップアプリです。Windows、macOS、Linux に対応し、アカウントは不要です。作業データはコンピューター内に保存されます。

## 主な機能

- **画面上の色を取得**：複数画面に対応したオーバーレイと、ポインターに追従する拡大鏡でピクセルを確認できます。取得した色は自動的にコピーされます。
- **ポートプロセスを管理**：個別のポートや範囲を検索し、確認後に選択したプロセスをまとめて終了できます。
- **スクリーンショットを撮影**：ディスプレイ全体または自由選択した範囲を撮影し、保存、クリップボードへのコピー、履歴への追加を自動で行います。
- **画面を録画**：ディスプレイ、選択範囲、アプリウィンドウを録画できます。ライブプレビュー、解像度、フレームレート、ビットレート、マイク音声に対応しています。
- **安全な文字列を生成**：英数字、数字、HEX、記号を含む文字列、UUID v4 を生成できます。
- **柔軟にカスタマイズ**：ライト／ダークテーマ、4 言語、カスタムフォント、グローバルショートカット、保存先、システムトレイ動作を設定できます。

## クイックスタート

1. ToolDock をインストールします。Linux では AppImage を直接起動することもできます。
2. 「設定」を開き、表示言語、フォント、スクリーンショットと録画の保存先を選びます。
3. サイドバーから使用するツールを選びます。
4. ToolDock をシステムトレイに常駐させるか、グローバルショートカットから機能を直接起動します。

アプリの表示言語は簡体字中国語、英語、日本語、韓国語に対応しています。

## カラーピッカー

1. 「カラーピッカー」を開き、「画面から色を取得」を選択します。
2. ToolDock のメインウィンドウが隠れ、すべてのディスプレイに半透明の暗いオーバーレイが表示されます。
3. ポインターを動かし、追従する拡大鏡でピクセルを確認します。
4. クリックして色を取得するか、`Esc` キーでキャンセルします。
5. 色の値は自動的にクリップボードへコピーされ、HEX と RGB は履歴から再度コピーできます。

複数ディスプレイでも拡大鏡は一つだけ表示され、現在のポインターを追従します。

## ポートプロセス管理

1. 「ポートプロセス」を開きます。
2. ポートをカンマまたはスペースで区切って入力します。`8000-8010` のような範囲も使用できます。
3. 「検索」を選択します。
4. プロセス名、PID、状態、起動コマンド、メモリ使用量を確認します。
5. 一つ以上の結果を選択し、「選択したプロセスを終了」を選んで確認します。

最近検索したポートは更新後も保持されます。保護されたプロセスや管理者権限で動作するプロセスを終了するには、ToolDock に同等の権限が必要です。終了前に PID を確認してください。

## スクリーンショット

1. 「スクリーンショット」を開きます。
2. 「ディスプレイ全体」または「範囲を選択」を選びます。
3. ディスプレイと、必要に応じて撮影までの待ち時間を選びます。
4. 範囲撮影では、暗いデスクトップオーバーレイ上をドラッグして領域を指定します。
5. PNG 画像が保存され、クリップボードにも自動的にコピーされます。他のアプリへすぐに貼り付けられます。
6. 操作欄の下にある履歴から、以前の画像を開けます。

既定の保存先は `Pictures/ToolDock` です。「設定」で変更できます。

## 画面録画

画面録画には [FFmpeg](https://ffmpeg.org/) が必要です。

1. FFmpeg をインストールし、`ffmpeg` が `PATH` から実行できることを確認します。
2. 「画面録画」を開きます。
3. ディスプレイ、選択範囲、またはアプリウィンドウを選びます。
4. 出力解像度、フレームレート、ビットレートを設定します。
5. 音声が必要な場合は「音声を録音」を有効にし、入力デバイスを選びます。
6. 「録画を開始」を選び、ライブプレビューで現在の映像を確認します。
7. 「停止して保存」を選びます。MP4 ファイルが録画履歴に追加され、ToolDock 内で再生できます。

既定の保存先は `Videos/ToolDock` です。「設定」で変更できます。FFmpeg が独自の場所にある場合は、ToolDock を起動する前に `TOOLDOCK_FFMPEG` を FFmpeg 実行ファイルのフルパスに設定してください。

ToolDock は次の順序で FFmpeg を検索します。

1. 環境変数 `TOOLDOCK_FFMPEG`
2. `PATH` 上の `ffmpeg`
3. アプリケーションの隣にある一般的な実行ファイルの場所

FFmpeg がなくても、色取得、ポート管理、スクリーンショット、文字列生成は使用できます。

## 文字列生成

1. 「文字列生成」を開きます。
2. 英数字、英字、数字、HEX、UUID v4 から種類を選びます。
3. 長さと生成数を設定します。
4. 必要に応じて記号を有効にします。
5. 生成後、一件またはすべての結果をコピーします。

## 設定とショートカット

「設定」では次の項目を変更できます。

- 表示言語とフォント
- スクリーンショットと録画の保存先
- グローバルショートカット
- 終了時の動作、またはシステムトレイへの常駐

既定のグローバルショートカット：

| 操作 | Windows / Linux | macOS |
| --- | --- | --- |
| 画面から色を取得 | `Ctrl+Alt+C` | `Command+Option+C` |
| 範囲スクリーンショット | `Ctrl+Alt+S` | `Command+Option+S` |
| 録画を開始または停止 | `Ctrl+Alt+R` | `Command+Option+R` |

ショートカットは「設定」で変更できます。各機能には異なるキーの組み合わせを設定してください。

## OS の権限

- **Windows**：WebView2 が必要です。管理者プロセスを終了する場合は、ToolDock も管理者として実行する必要があります。
- **macOS**：色取得、スクリーンショット、録画には「画面収録」の権限が必要です。色取得には「入力監視」、音声録音にはマイクの権限が必要になる場合があります。
- **Linux**：X11 で最も広くキャプチャ機能を利用できます。Wayland での動作はコンポジターとデスクトップポータルに依存します。録画には PipeWire と適切な権限が必要になる場合があります。

## トラブルシューティング

**FFmpeg が見つからないと表示される**

ターミナルで `ffmpeg -version` を実行してください。コマンドを実行できない場合は、FFmpeg を `PATH` に追加するか `TOOLDOCK_FFMPEG` を設定し、ToolDock を再起動します。

**グローバルショートカットが反応しない**

「設定」で別の組み合わせを選んでください。同じショートカットが OS や他のアプリで使われている可能性があります。

**プロセスを終了できない**

対象が保護されているか、より高い権限で動作している可能性があります。ToolDock を同等の権限で再起動し、PID を確認してから再試行してください。

**macOS または Linux でキャプチャ機能が動作しない**

OS の画面キャプチャ権限を確認してください。Wayland ではデスクトップ環境とポータル設定によって対応状況が異なります。

## プライバシー

ToolDock は、スクリーンショット、録画、色、プロセス情報、生成した文字列をローカルで処理します。アカウントは不要で、これらのデータを ToolDock のサービスへアップロードしません。

サイドバーには RouteMarket.ai と RouteMarket Tools への任意のリンクがあります。リンクは UTM パラメーター付きで既定のブラウザーに開かれますが、ToolDock 内のデータは送信されません。
