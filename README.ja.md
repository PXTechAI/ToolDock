# ToolDock

[English](README.md) | [简体中文](README.zh-CN.md) | **日本語**

ToolDock は、Windows、macOS、Linux で動作するローカル開発者向けデスクトップツールです。アカウントやバックグラウンドサービスは必要ありません。

## 機能

- **スクリーンカラーピッカー**：全ディスプレイのオーバーレイと、カーソルに追従する単一の拡大鏡で色を取得します。値は自動的にコピーされ、`Esc` でキャンセルできます。
- **ポートとプロセスの管理**：ポートを使用しているプロセスを確認し、PID、コマンド、メモリ情報を表示して、確認後に複数のプロセスを終了できます。
- **スクリーンショット**：ディスプレイ全体または自由選択した領域を PNG として保存し、クリップボードへ自動コピーして履歴に表示します。
- **画面録画**：ディスプレイ、選択領域、またはアプリウィンドウを録画し、リアルタイムプレビューと録画履歴を表示します。
- **安全な文字列生成**：指定した長さのランダム文字列、HEX、数字、記号、UUID v4 を生成できます。
- **外観と保存先の設定**：サイドバーでテーマを切り替え、保存先、グローバルショートカット、システムトレイ動作を設定できます。

サイドバーには RouteMarket.ai と RouteMarket Tools への任意のリンクもあります。リンクは UTM パラメータ付きでシステムブラウザーに開かれます。ToolDock のスクリーンショット、録画、色、プロセス情報は送信されません。

## インストール方式

GitHub Releases では、Windows と macOS に標準インストーラーを提供し、Linux にはインストール版とポータブル版の両方を提供する構成を推奨します。

| プラットフォーム | パッケージ | 推奨 |
| --- | --- | --- |
| Windows x64 | NSIS `.exe` | 推奨。スタートメニューと標準のアンインストール項目を作成します。 |
| macOS Apple Silicon | `.dmg` | M1 以降の Mac 向けです。 |
| macOS Intel | `.dmg` | Intel Mac 向けです。 |
| Linux x64 | `.AppImage` | インストール不要のポータブル版です。 |
| Linux x64 | `.deb` | Debian、Ubuntu 系向けのインストール版です。 |

Windows/macOS の一般的な操作感を保ちつつ、Linux ではポータブル実行も選択できます。初期の公開ビルドが未署名の場合、OS のセキュリティ警告が表示されることがあります。

## 使い方

### 画面から色を取得する

1. 「取色器」を開きます。
2. 「从屏幕取色」を選択します。
3. ToolDock のウィンドウが一時的に非表示になり、デスクトップオーバーレイと拡大鏡が表示されます。
4. 画面上のピクセルをクリックして取得するか、`Esc` でキャンセルします。
5. 値は自動的にコピーされ、結果パネルから再度 HEX または RGB をコピーできます。

### ポートを確認してプロセスを終了する

1. 「端口进程」を開きます。
2. カンマまたはスペース区切りでポートを入力します。`8000-8010` のような範囲も使用できます。
3. 「查询」を選択します。
4. プロセス名、PID、状態、コマンド、メモリ使用量を確認します。
5. 対象を選択し、「结束所选」を選択して確認します。

保護されたプロセスや管理者権限で動作するプロセスを終了するには、ToolDock に同等の権限が必要です。終了前に PID を確認してください。

### スクリーンショットを取得する

1. 「截图」を開きます。
2. 「完整显示器」または「选择区域」を選択します。
3. ディスプレイと必要に応じて遅延時間を選択します。
4. 範囲指定ではアプリが非表示になり、すべてのディスプレイにオーバーレイが表示されます。ドラッグして矩形領域を選択します。
5. 保存した画像はクリップボードに自動コピーされ、他のアプリへ直接貼り付けられます。
6. 画面下部の履歴から最近のスクリーンショットを確認できます。

既定の保存先は `Pictures/ToolDock` です。「设置」から変更できます。

### 画面を録画する

1. FFmpeg をインストールし、`ffmpeg` が `PATH` から実行できることを確認します。
2. 「屏幕录制」を開きます。
3. ディスプレイ、選択領域、またはアプリウィンドウを選び、出力解像度、フレームレート、ビットレートを設定します。
4. 「开始录制」を選択すると、左側にリアルタイムのキャプチャ映像が表示されます。
5. 完了時に「停止并保存」を選択すると、結果が画面下部の録画履歴に追加されます。

既定の保存先は `Videos/ToolDock` です。「设置」から変更できます。FFmpeg が独自の場所にある場合は、ToolDock の起動前に `TOOLDOCK_FFMPEG` を FFmpeg 実行ファイルのフルパスに設定してください。

### 文字列を生成する

1. 「字符串生成」を開きます。
2. 英数字、英字、数字、HEX、UUID v4 から種類を選択します。
3. 長さと生成数を設定します。
4. 必要に応じて記号を有効にします。
5. 生成後、個別またはすべての結果をコピーします。

## 設定

左下のテーマボタンからダーク/ライトモードをすぐに切り替えられます。選択したテーマは次回起動時にも保持されます。

「设置」では次の項目を設定できます。

- スクリーンショットの保存先と履歴の参照先。
- 画面録画の保存先と履歴の参照先。
- 色取得、範囲スクリーンショット、録画の開始/停止に使うグローバルショートカット。
- 閉じると終了するか、システムトレイへ格納するか。

存在しないフォルダーは、メディア保存時に自動で作成されます。

## OS の権限

- **Windows**：WebView2 が必要です。管理者プロセスを終了する場合、ToolDock も管理者として実行する必要があります。
- **macOS**：色取得、スクリーンショット、録画には「画面収録」権限が必要です。色取得には「入力監視」権限も必要になる場合があります。
- **Linux**：X11 が最も広く対応しています。Wayland の動作はコンポジターとデスクトップポータルに依存します。録画には PipeWire も必要です。

## FFmpeg

画面録画では外部の FFmpeg を使用して H.264 をエンコードします。配布サイズの大幅な増加とライセンス確認を避けるため、FFmpeg はアプリに同梱していません。

ToolDock は次の順序で FFmpeg を検索します。

1. `TOOLDOCK_FFMPEG`
2. `PATH` 上の `ffmpeg`
3. アプリケーションの隣にある一般的な実行ファイルの場所

FFmpeg がなくても、スクリーンショット、色取得、ポート管理、文字列生成は使用できます。

## 開発

必要な環境：

- Node.js 22
- Rust stable
- 各 OS 向けの Tauri 2 システム依存関係
- 録画をテストする場合は FFmpeg

```bash
npm ci
npm run desktop:dev
```

ブラウザーのデモモードで UI のみを起動する場合：

```bash
npm run dev
```

すべてのローカルチェックを実行する場合：

```bash
npm run check
```

現在の OS 向けデスクトップパッケージをビルドする場合：

```bash
npm run desktop:build
```

ネイティブパッケージは対象 OS 上でビルドする必要があります。

## CI とリリース

- `.github/workflows/ci.yml` はバージョン確認、フロントエンドビルド、Rust フォーマット確認、Windows/macOS/Linux のネイティブチェックを実行します。
- `.github/workflows/release.yml` は Windows NSIS、macOS DMG、Linux AppImage、Linux DEB をビルドします。
- `.github/workflows/pages.yml` は `website/` の多言語プロジェクトサイトを GitHub Pages に公開します。
- `v0.2.0` のようなタグを push すると Draft GitHub Release が作成されます。成果物を確認してから手動で公開してください。

リリース前に `package.json`、`src-tauri/tauri.conf.json`、`src-tauri/Cargo.toml` のバージョンを一致させ、`CHANGELOG.md` を更新してください。

詳細は[リリースガイド](docs/RELEASING.md)を参照してください。
