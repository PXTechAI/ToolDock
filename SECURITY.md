# Security Policy

ToolDock can inspect local ports, terminate processes, capture displays, and read screen pixels. Security reports should not be posted as public issues.

Use the repository's **Security > Report a vulnerability** feature to submit a private report. Include the affected version, operating system, reproduction steps, and impact. Remove credentials and unrelated personal data from logs and screenshots.

The project does not silently elevate privileges, run a background service, or send captured data to a remote server.
